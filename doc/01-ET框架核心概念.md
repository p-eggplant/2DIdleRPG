# ET 框架核心概念

## 一、ECS 架构

ET 框架基于 **ECS（Entity-Component-System）** 架构，这是理解整个项目的基础。

| 概念 | 作用 | 对应本项目的目录 | 类比 |
|---|---|---|---|
| **Entity** | 只是一个"空壳子"，用来挂组件的 ID 容器 | 无需手动创建，框架自动管理 | 一个空盒子 |
| **Component** | 纯数据，不包含逻辑 | `Server/Model/`、`Unity/Codes/Model/` | 盒子里的卡片 |
| **System** | 静态扩展方法，操作 Component 的数据 | `Server/Hotfix/`、`Unity/Codes/Hotfix/` | 操作卡片的手 |

### 典型例子：PlayerRoleUpComponent

```csharp
// === Component (Model层) - 只有数据 ===
public class PlayerRoleUpComponent : Entity, IAwake, IDestroy, IUnitCacheNode
{
    public int m_CurLevel = 0;  // 纯数据
}

// === System (Hotfix层) - 操作数据 ===
public static class PlayerRoleUpComponentSystem
{
    public static int LevelUp(this PlayerRoleUpComponent self)
    {
        self.m_CurLevel++;  // 修改数据
        return self.m_CurLevel;
    }
}
```

## 二、项目分层（4 层架构）

### 服务端（Server/）

```
Model 层（数据定义层）
  ├── 存放：Component、Config 类、Message 类
  ├── 特点：只存数据，没有逻辑
  └── 目录：Server/Model/

Hotfix 层（业务逻辑层）
  ├── 存放：System（扩展方法）、Handler（网络处理）、Event（生命周期）
  ├── 特点：所有逻辑都在这里
  └── 目录：Server/Hotfix/
```

### 客户端（Unity/Codes/）

```
Model 层        → 客户端数据定义（服务端的镜像）
ModelView 层    → UI 组件定义（什么 UI 窗口、有什么控件）
Hotfix 层       → 客户端业务逻辑（发消息、处理回复）
HotfixView 层   → UI 业务逻辑（加载预制体、按钮事件、刷新界面）
Mono 层         → 不热更的基础代码（Init、CodeLoader）
```

**重要**：不要搞混 `Model` 和 `ModelView`
- `Model` = 游戏数据（等级、背包、金币）
- `ModelView` = UI 数据（窗口引用、控件引用）

## 三、组件的生命周期（ComponentBase 事件系统）

每个 Component 都通过一个 Event 类来管理其生命周期：

```csharp
[PlayerComponentEvent(nameof(PlayerRoleUpComponent))]
public class PlayerRoleUpComponentEvent : ComponentBase<PlayerRoleUpComponent>
{
    // 1. 组件挂载到 Player 上
    OnCreateComponent(ref Player)

    // 2. 新玩家第一次创建（无存档时）
    OnFirstCreate(self)          → 设置默认值

    // 3. 从数据库加载数据（有存档时）
    OnImprotDBData(self, db)     → 恢复 m_CurLevel

    // 4. 数据初始化完毕后计算
    OnInit(self)                 → 计算属性值

    // 5. 登出时导出数据到 DB
    OnExportDBData(self)

    // 6. 登录时导出数据给客户端
    OnExportLoginData(self, ref info)
}
```

**执行顺序**：`CreateComponent` → `FirstCreate/ImprotDBData` → `Init` → （游戏中修改数据） → `ExportDBData` → `ExportLoginData`

## 四、关键术语速记

| 术语 | 通俗解释 |
|---|---|
| **Entity** | 空盒子，用来装 Component |
| **Component** | 数据卡片，比如 `m_CurLevel` |
| **System** | 操作卡片的手，比如 `LevelUp()` |
| **AMActorRpcHandler** | 服务端处理请求的基类（收到请求→处理→回复） |
| **IUnitCacheNode** | 给组件打标记："这个组件需要存盘" |
| **Proto** | 客户端与服务端通信的消息格式定义 |
| **FriendClass** | 允许跨程序集访问 private 字段的标记（ET 特有） |
| **Hotfix** | 热更层（可以不停服更新的代码） |
| **Model** | 不热更层（基础数据定义） |
| **RPC** | 远程过程调用（客户端发请求，服务端处理并回复） |
