# ET6 资源管理模块技术文档

> 基于 ET6 框架的客户端资源加载、管理、释放完整技术文档

## 1. 概述

### 1.1 核心架构

```
┌─────────────────────────────────────────────────────────┐
│                  ResourcesComponent                      │
│              (全局单例, Game.Scene 上)                    │
│  ┌──────────┐  ┌──────────────┐  ┌──────────────────┐   │
│  │ bundles  │  │ resourceCache │  │ 依赖解析(拓扑排序) │   │
│  │(ABInfo)  │  │(bundle→assets)│  │ DependenciesCache │   │
│  └──────────┘  └──────────────┘  └──────────────────┘   │
│  ┌──────────────────────────────────────────────────┐    │
│  │           AssetBundleManifest                     │    │
│  │       (从 StreamingAssets AB 加载)                 │    │
│  └──────────────────────────────────────────────────┘    │
└───────────────────────┬─────────────────────────────────┘
                        │
        ┌───────────────┼───────────────┐
        ▼               ▼               ▼
┌───────────┐   ┌───────────┐   ┌───────────┐
│ResourcesLoader│ │UIComponent│   │ConfigLoader│
│  Component   │ │ (窗口管理) │   │(配置加载)   │
│(场景级追踪)   │ │           │   │           │
└───────────┘   └───────────┘   └───────────┘
```

### 1.2 设计理念

| 机制 | 说明 |
|---|---|
| **引用计数** | 每个资源包 ABInfo 维护 RefCount，加载+1，卸载-1，RefCount ≤ 0 时真卸载 |
| **依赖解析** | 通过 AssetBundleManifest 递归收集依赖，拓扑排序确保加载/卸载顺序正确 |
| **协程锁** | CoroutineLockType.Resources(6) / ResourcesLoader(7) 防止并发加载同一资源 |
| **双路径适配** | Editor 模式用 AssetDatabase 直读，Runtime 模式用 AssetBundle.LoadFromFile |
| **分层架构** | Model(数据定义) → ModelView(Unity 引用) → Hotfix(逻辑) → HotfixView(UI/渲染) |

## 3. 美术资源管理

### 3.1 Unit（游戏对象）创建流程

```
服务端下发 UnitInfo
│
├─1. UnitFactory.Create(currentScene, unitInfo)
│      └─ 创建 Unit Entity，设置 Position/Rotation
│      └─ 添加 NumericComponent / MoveComponent / ObjectWait
│      └─ 发布 EventType.AfterUnitCreate
│
└─2. AfterUnitCreate_CreateUnitView.Run()
       a) ResourcesComponent.Instance.GetAsset("Unit.unity3d", "Unit")
          从缓存中获取 Unit 预设体 Assets 中的根 GameObject
       b) bundleGameObject.Get<GameObject>("Skeleton")
          通过 ReferenceCollector 获取 "Skeleton" 子对象
       c) Object.Instantiate(prefab, GlobalComponent.Instance.Unit, true)
          在 /Global/Unit 节点下实例化
       d) 设置 Position、添加 GameObjectComponent、AnimatorComponent
```

### 3.2 ReferenceCollector 机制

这是项目中 Art 资源引用的核心组件：

- 在 Prefab 上挂载 `ReferenceCollector` 组件，以 Key-Value 形式管理子资源的引用
- 代码中通过 `GameObjectHelper.Get<T>(key)` 获取指定子对象
- 避免了 `GameObject.Find()` 字符串查找的性能开销

```csharp
// 使用示例 (GameObjectHelper.cs)
public static T Get<T>(this GameObject gameObject, string key) where T : class
{
    return gameObject.GetComponent<ReferenceCollector>().Get<T>(key);
}

// 实际调用 (AfterUnitCreate_CreateUnitView.cs)
GameObject prefab = bundleGameObject.Get<GameObject>("Skeleton");
```

### 3.3 动画管理（AnimatorComponent）

```
AnimatorComponent (挂载在 Unit Entity 上)
│
├─ Awake: 从 GameObject 获取 Animator，缓存 clips + 参数名
├─ Update: 每帧应用 MotionType 触发器 + MotionSpeed 浮动值
│
├─ Play(MotionType, speed): 播放指定动画

├─ SetBool/Float/Int/Trigger: 参数控制

└─ MotionType 枚举: None, Idle, Run
```

## 4. UI 资源管理

### 4.1 UI 分层架构

```
/Global/UI
├── Low (低层)
├── Mid (中层)
├── High (高层)
└── Tips (提示层，最前)
└── ItemTemp (Item 缓存池)
```

通过 `EUILayer` 枚举管理窗口显示层级。

### 4.2 窗口生命周期（5阶段）

```
1. CreateWindow (系统初始化时)
   └─ UIComponent 启动时扫描所有 [UIEvent] 标记的类
   └─ 创建 IUIEvent 实例并注册到 m_dicEvent 字典
   └─ 调用 OnCreateWindow() → 给 UIComponent 添加窗口 Entity 组件

2. LoadResouse (首次 ShowWindow 时)
   └─ ResourcesLoaderComponent.Load("xxx.unity3d") 加载 AB
   └─ ResourcesComponent.GetAsset(abName, prefabName) 获取 Prefab
   └─ GameObject.Instantiate(prefab, layer) 实例化到对应层级
   └─ 通过 ReferenceCollector 获取 UI 控件引用，注册事件

3. ShowWindow (每次显示时)
   └─ SetActive(true)
   └─ 调用 OnShowWindow(showData) 刷新 UI 数据

4. HideWindow (隐藏但不销毁)
   └─ SetActive(false)
   └─ 调用 OnHideWindow() 清理定时器等

5. DestroyResouse (销毁窗口)
   └─ 销毁所有子对象
   └─ 销毁窗口 GameObject
```

### 4.3 UI 预制体池化（Item 缓存）

```csharp
// 缓存
UIComponent.Instance.ItemHold(itemGameObject);
// → 设置父节点为 /Global/UI/ItemTemp，SetActive(false)

// 获取
GameObject item = UIComponent.Instance.ItemGet(itemName);
// → 从缓存中取出并激活（减少 Instantiate/Destroy 开销）

// 销毁
UIComponent.Instance.ItemDestory(itemName);
```

### 4.4 异步锁机制

防止按钮快速点击导致重复操作：

```csharp
// 带锁的按钮监听 (UIHelper)
public static async void AddListenerAsync(this Button button, Func<ETTask> action)
{
    button.onClick.AddListener(async () =>
    {
        UIComponent.Instance.AddLock();   // m_AsyncLock++
        using (await CoroutineLockComponent.Instance.Wait(...))
        {
            await action();
        }
        UIComponent.Instance.UnLock();    // m_AsyncLock--
    });
}
```

## 6. 资源加载核心机制

### 6.1 ResourcesComponent 核心数据结构

```csharp
// 全局 AB 缓存: bundleName → ABInfo
Dictionary<string, ABInfo> bundles;

// 资源缓存: bundleName → (assetName → UnityEngine.Object)
Dictionary<string, Dictionary<string, UnityEngine.Object>> resourceCache;

// 依赖缓存: bundleName → 排序后的依赖列表
Dictionary<string, string[]> DependenciesCache;

// AB 名称小写映射
Dictionary<string, string> BundleNameToLowerDict;

// 每个 AB 的元信息
class ABInfo {
    string Name;           // 包名
    int RefCount;          // 引用计数
    AssetBundle AssetBundle; // Unity AB 实例
    bool AlreadyLoadAssets;  // 是否已加载全部资源
}
```

### 6.2 加载流程

```
ResourcesComponent.LoadBundleAsync(abName)
│
├─1. 依赖解析
│   └─ GetSortedDependencies(abName)
│       ├─ 递归遍历 AssetBundleManifest.GetAllDependencies()
│       ├─ 删除已加载的依赖
│       ├─ 拓扑排序（被依赖最多的排在最后加载）
│       └─ 检测循环依赖（抛出异常）
│
├─2. 逐依赖加载
│   └─ LoadOneBundleAsync(depAbName)
│       ├─ 已加载 → RefCount++
│       ├─ Editor 模式(!IsAsync && IsEditor)
│       │   └─ AssetDatabase.GetAssetPathsFromAssetBundle()
│       │       → 逐文件 AssetDatabase.LoadAssetAtPath()
│       │       → 缓存到 resourceCache[abName][assetName]
│       └─ Runtime 模式
│           ├─ 优先从 PersistentDataPath 加载（热更路径）
│           ├─ 回退到 StreamingAssets（内置路径）
│           ├─ AssetBundle.LoadFromFileAsync(path)
│           ├─ LoadAllAssetsAsync() 加载全部资源
│           └─ 逐资源缓存到 resourceCache[abName][assetName]
│
├─3. 获取资源
│   └─ GetAsset(abName, assetName)
│       └─ return resourceCache[abName][assetName]
```

### 6.3 卸载流程

```csharp
ResourcesComponent.UnloadBundleAsync(abName)
│
├─ 获取排序后的依赖列表（拓扑反序）
├─ 逐帧卸载（每帧卸载一个，避免卡顿）
│   └─ RefCount--
│       └─ RefCount <= 0 时：
│           ├─ resourceCache.Remove(bundleName)
│           ├─ ABInfo.AssetBundle.Unload(true)  // 卸载 AB 及所有加载的资源
│           ├─ ABInfo.Dispose()
│           └─ bundles.Remove(bundleName)
```

### 6.4 ResourcesLoaderComponent（场景级追踪）

```csharp
// 挂载在 ZoneScene 上
class ResourcesLoaderComponent {
    HashSet<string> LoadedResource;   // 记录本场景加载了哪些 AB
}

// 场景销毁时
OnDestroy:
    await 等 5 秒（防止场景快速重建导致的不必要卸载）
    foreach (ab in LoadedResource)
        ResourcesComponent.Instance.UnloadBundleAsync(ab);
```

> **5秒延迟的意义**：场景切换时如果新旧场景同名，立即卸载会导致资源重新加载。

### 6.5 协程锁防并发

```
CoroutineLockType.Resources = 6      → LoadBundleAsync 使用
CoroutineLockType.ResourcesLoader = 7 → ResourcesLoader.LoadAsync 使用

加锁粒度：按 AB 名称的 HashCode
超时时间：60000ms
```

### 6.6 Editor vs Runtime 双路径

| | Editor 模式 | Runtime 模式 |
|---|---|---|
| 条件 | `!Define.IsAsync && Define.IsEditor` | `Define.IsAsync \|\| !Define.IsEditor` |
| 加载方式 | `AssetDatabase.LoadAssetAtPath()` | `AssetBundle.LoadFromFile()` |
| 优势 | 无需打包 AB，开发效率高 | 真实模拟发布环境 |
| 切换方式 | `Define.IsAsync` 编译开关 | 打包后自动走 Runtime |

## 7. 音频视频资源（现状与处理建议）

### 7.1 现状分析

- 当前项目仅有一个音效文件：`Bundles/sound/sound_updata_open.MP3`
- 没有独立的音频/视频管理组件或 System
- 音效文件会被打包成 `sound.unity3d` AB

### 7.2 音频资源集成建议

如需增加音频播放功能，按以下步骤：

```
1. 创建 SoundComponent (ModelView/Demo/Sound/)
   ├─ 持有 AudioSource 引用
   ├─ 管理音效 Clip 缓存字典
   └─ 提供 Play(abName, clipName, volume) 接口

2. 创建 SoundComponentSystem (HotfixView/Demo/Sound/)
   ├─ Awake: 在 Global/ 下创建 AudioSource 对象
   ├─ PlayBGM / PlaySFX: 从 AB 加载 AudioClip 并播放
   └─ 音量控制

3. 新增 AB: sound.unity3d（已存在）
   内部按类型组织: bgm/xxx, sfx/xxx

4. 注册为 ObjectSystem，挂载到场景 Entity 上
```

### 7.3 视频资源集成建议

目前项目无视频需求。如需支持，建议：

```
1. 引入 Unity VideoPlayer 组件
2. 创建 VideoComponent 管理视频播放
3. 视频资源通过 ResourcesComponent.GetAsset(abName, videoName) 获取
   返回 VideoClip，通过 VideoPlayer.url 直接播放
4. 注意：视频文件不宜打进普通 AB，建议使用 StreamingAssets 直接路径加载
```

## 8. 总结

### 8.1 优点

| 优点 | 说明 |
|---|---|
| **引用计数防泄漏** | 通过 ABInfo.RefCount 精确追踪，RefCount ≤ 0 才真正释放，不会过早/过晚卸载 |
| **协程锁防并发** | 同一 AB 不会被同时加载多次，避免资源重复占用内存 |
| **双路径开发友好** | Editor 模式免打包直接运行，Runtime 模式真实模拟，开发效率高 |
| **ECS 解耦清晰** | Model 层不依赖 UnityEngine，ModelView 层持有引用，热更层可安全重载 |
| **场景级资源追踪** | ResourcesLoaderComponent 记录每个场景加载的 AB，场景切换时自动释放 |
| **UI 窗口池化** | ItemHold/ItemGet 缓存机制减少频繁 Instantiate/Destroy 开销 |
| **配置表驱动** | Excel → protobuf .bytes → C# 类，策划可直接修改配置 |

### 8.2 缺点与改进方向

| 缺点 | 说明 | 改进建议 |
|---|---|---|
| **加载非真正异步** | LoadBundleAsync 实际是协程 + 同步 LoadAllAssets，不是多线程 | 对大体积 AB 可考虑分帧加载 |
| **无 Streaming/LOD** | 所有资源一次性全量加载，大世界场景可能内存过高 | 引入 AssetBundle 粒度拆分 + 按需加载 |
| **热更依赖手动管理** | AB 打包需手动配置依赖关系，新增资源时容易遗漏 | 使用 Unity AssetBundle Browser 或 Addressables |
| **无音视频管理** | 缺少 SoundComponent / VideoComponent，不支持多媒体播放 | 按第7章建议扩展 |
| **Editor 模式不校验 AB 完整性** | Editor 直读 AssetDatabase，可能遗漏 AB 打包配置错误 | 增加 CI 检查环节 |
| **5秒卸载延迟** | 场景切换后旧资源仍占用内存5秒 | 可通过事件通知立即卸载 |

### 8.3 注意事项

1. **AB 名称必须小写**：`ResourcesComponent` 内部通过 `BundleNameToLowerDict` 做大小写映射，所有 AB 名在查找前会转小写。自定义 AB 命名时不会有问题，但查找时需保持一致。

2. **卸载必须走 ResourcesLoaderComponent**：不要在业务代码中直接调用 `ResourcesComponent.UnloadBundle()`，应由场景级的 `ResourcesLoaderComponent` 统一管理生命周期。

3. **GameObject 不要跨场景使用**：Instantiate 的 GameObject 通过 `ResourcesLoaderComponent` 关联了 AB 生命周期。如果跨场景引用，AB 卸载后 GameObject 上的资源引用会失效。

4. **UI 窗口不要直接 Destroy**：使用 `UIComponent.HideWindow()` 或 `HideWindowDestroy()` 管理窗口生命周期，直接 Destroy 会导致 UIComponent 状态不一致。

5. **Editor 模式注意资源路径**：Editor 模式下 `AssetDatabase.LoadAssetAtPath` 使用的是 `Assets/Bundles/` 路径下的原始文件，而非打包后的 AB。如果资源不在正确目录，Editor 模式会加载失败。

6. **协程锁超时**：当前锁超时设置为 60000ms，如果某个 AB 加载超过60秒会触发超时异常，注意大资源包的加载时间。

7. **热更 DLL 的特殊性**：`code.unity3d` 和 `config.unity3d` 是框架层加载的，不走 `ResourcesComponent` 系统，由 `CodeLoader` 和 `ConfigLoader` 直接管理。

### 8.4 新增资源类型指南

当需要新增一种资源类型时，按以下步骤操作：

```
1. 将资源放入 Bundles/ 对应目录
   ├─ 场景资源 → Bundles/ 根目录或 Scenes/ 子目录
   ├─ UI 预制体 → Bundles/dialog/{模块名}/ 或 Bundles/UI/
   ├─ 角色模型 → Bundles/Character/
   ├─ 特效 → Bundles/Effect/
   ├─ 音效 → Bundles/sound/
   └─ 配置 → Bundles/Config/

2. Unity Editor 中设置 Asset Bundle Name
   格式: {资源名小写}.unity3d（extension 可在 ResourcesComponent 中配置）

3. 创建 ECS 组件定义
   ModelView 层: 定义持有 UnityEngine.Object 引用的 Component
   HotfixView 层: 编写 System 处理加载/卸载逻辑

4. 使用 ResourcesLoaderComponent 管理生命周期
   scene.GetComponent<ResourcesLoaderComponent>().LoadAsync("xxx.unity3d")

5. 通过 ResourcesComponent 获取资源
   ResourcesComponent.Instance.GetAsset(abName, assetName)

6. 注册 [ObjectSystem] 确保框架能扫描到新类
```

### 8.5 资源目录结构参考

```
Assets/Bundles/
├── Code/                    → code.unity3d (热更 DLL)
├── Config/                  → config.unity3d (配置表)
├── Unit/                    → unit.unity3d (角色预设体)
├── Map/                     → map1.unity3d, map2.unity3d (场景)
├── UI/                      → uilobby.unity3d, uihelp.unity3d
├── dialog/                  → uilogin.unity3d (包含登录相关所有 UI)
├── sound/                   → sound.unity3d (音效)
├── Effect/                  → effect.unity3d (特效，预留)
├── Character/               → character.unity3d (角色模型，预留)
└── Shader/                  → shader 文件 (不打包 AB，单独处理)
```
