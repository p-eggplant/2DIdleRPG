## 一、常用 API 速查

### 获取数据

```csharp
// 从场景获取玩家对象
Player pPlayer = zoneScene.GetComponent<Player>();

// 从玩家获取某个组件
PlayerRoleUpComponent pRoleUp = pPlayer.GetComponent<PlayerRoleUpComponent>();
PlayerBagComponent pBag = pPlayer.GetComponent<PlayerBagComponent>();
PlayerDataComponent pData = pPlayer.GetComponent<PlayerDataComponent>();

// 从 UI 窗口获取场景（DlgRoleUp 继承 Entity）
Scene zoneScene = self.DomainScene();
```

### 网络通信

```csharp
// 获取 Session
SessionComponent pSession = zoneScene.GetComponent<SessionComponent>();

// RPC 调用（发请求，等回复）
Game2C_SomeResponse resp = (Game2C_SomeResponse)await pSession.Session.Call(
    new C2Game_SomeRequest());

// 检查是否成功
if (resp.Error != ErrorCode.ERR_Success)
{
    UITools.Tips($"失败: {resp.Message}");
    return;
}
```

## 二、控件命名规范

项目中的 UI 控件使用统一的前缀命名：

| 前缀 | 含义 | 示例 |
|---|---|---|
| `E` | Editor 引用（可被代码查找） | `EBt_Back`、`ELb_LevelNumb` |
| `Bt_` | Button 按钮 | `EBt_LevelUp`、`Bt_Material1` |
| `Lb_` / `ELb_` | Label/Text 文本 | `ELb_LevelNumb`、`Lb_Atk` |
| `Img_` / `EImg_` | Image 图片 | `Img_BG`、`EImg_MaterialIcon1` |
| `Panel_` | 容器面板 | `Panel_Up`、`Panel_Material` |

## 三、常见错误及解决

### 1. "当前上下文中不存在名称 XXX"

**原因**：要么没加 `using`，要么没写命名空间，要么跨程序集访问私有字段。

**解决**：
- 检查 `using UnityEngine;` 等引用
- 检查类是否在 `namespace ET` 内
- 访问其他组件的字段时加 `[FriendClass(typeof(XXX))]`

### 2. 新文件在 VS 中显示为"杂项文件"

**原因**：Unity 生成的 `.csproj` 使用了逐文件列举，新增文件未被包含。

**解决**：
- 双击运行项目根目录下的 `刷新项目文件.bat`
- 然后在 VS 中右键项目 → 重新加载
- 或安装 `CsprojWildcardFixer`（已将 `.csproj` 改为通配符模式）

### 3. 客户端修改数据后服务端不生效

**原因**：客户端只改了本地数据，没发 RPC 请求到服务端。

**解决**：所有数据修改都应通过服务端 Handler 处理，客户端仅做校验和 UI 展示。

### 4. 数据升级后重启游戏恢复原样

**原因**：升级后没调 `SetOne()` 或 `Set()` 存盘，或存盘路径没走对。

**解决**：在 Handler 中增删改数据后调用 `SetOne()` 即时存盘。

## 五、开发守则

1. **Model 层只放数据**，不放逻辑
2. **Hotfix 层放逻辑**，不放数据定义
3. **不要直接修改 Proto 生成的文件**，改 `.proto` 源文件后重新导出
4. **不要直接修改 Config 生成的文件**，改 Excel 后重新导出
5. **Handler 中 `reply()` 必须调用**，否则客户端永远等不到回复
6. **网络请求用 `await`**，不要用回调，否则容易造成回调地狱
7. **数据增删改后要存盘**，调用 `SetOne()` 或 `Set()`
