# ET6 UI 框架技术文档

> 基于本项目实现的 UI 框架设计，采用 **三层架构 + 事件驱动 + 资源按需加载** 的模式。

---

## 一、架构总览

### 三层结构

```
┌─────────────────────────────────────────────────┐
│                   ModelView 层                   │
│           UI组件定义（Entity + 控件引用）          │
│   DlgRoleUp.cs, UIPackage.cs, UITips.cs ...      │
├─────────────────────────────────────────────────┤
│                  HotfixView 层                    │
│         UI生命周期 + 业务逻辑（System + Event）    │
│   DlgRoleUpEvent.cs（加载/销毁/显示/隐藏）        │
│   DlgRoleUpSystem.cs（按钮回调/刷新/网络请求）     │
├─────────────────────────────────────────────────┤
│                 Prefab 资源层                     │
│        Unity 预制体（.prefab） + AssetBundle      │
│   Bundles/dialog/dlg_role/dlg/dlg_role_up.prefab │
└─────────────────────────────────────────────────┘
```

### 核心职责

| 层 | 目录 | 职责 | 能否热更 |
|---|---|---|---|
| **ModelView** | `Codes/ModelView/` | UI Entity 定义、控件引用、UI 框架核心类 | ❌ 不能 |
| **HotfixView** | `Codes/HotfixView/` | 窗口生命周期、按钮回调、网络请求、刷新逻辑 | ✅ 能 |
| **Prefab** | `Assets/Bundles/` | 界面布局、图片文字、动画 | ✅ 资源热更 |

## 二、核心类说明

### 2.1 层级定义（EUILayer）

```csharp
public enum EUILayer
{
    low,    // 底层（主界面背景）
    mid,    // 中层（主界面内容）
    high,   // 高层（弹窗对话框）
    tips,   // 提示层（Toast 提示、Loading）
    Max,    // 计数
}
```

**对应场景层级**：`/Global/UI/low`、`/Global/UI/mid`、`/Global/UI/high`、`/Global/UI/tips`

### 2.2 UIComponent（UI 管理器）

**单例**，挂载在 `Scene` 上，管理所有窗口的注册、加载、显示/隐藏、销毁。

```csharp
[ComponentOf(typeof(Scene))]
public class UIComponent : Entity, IAwake, IDestroy
{
    public static UIComponent Instance;                     // 单例
    public Dictionary<EUILayer, Transform> m_dicLayer;      // 4层UI根节点
    public Dictionary<string, IUIEvent> m_dicEvent;         // 窗口名 → 事件处理类
    public GameObject m_ItemParent;                         // 列表Item缓存挂载点
    public uint m_AsyncLock;                                // 异步操作锁
}
```

**关键方法**：

| 方法 | 说明 |
|---|---|
| `ShowWindow(szName, showData?)` | 显示窗口（如已加载则直接显示，否则先加载再显示） |
| `HideWindow(szName)` | 隐藏窗口（保留预制体在场景中） |
| `HideWindowDestroy(szName)` | 隐藏并销毁窗口（释放资源） |
| `DestroyAllWindow()` | 切换场景时销毁所有窗口 |
| `GetWindow<T>()` | 获取已打开的窗口实例 |
| `ItemHold(obj)` | 缓存列表 Item（用于列表循环利用） |
| `ItemGet(szName)` | 获取缓存的 Item 实例 |
| `ItemDestory(szName)` | 销毁缓存的 Item |

### 2.3 UIEventBase（窗口生命周期基类）

每个窗口对应一个继承 `UIEventBase<T>` 的类，实现 5 个生命周期方法：

```csharp
public abstract class UIEventBase<T> : IUIEvent where T : Entity
{
    // 1. 系统初始化时注册窗口组件
    abstract void OnCreateWindow(ref UIComponent pUIComponent);

    // 2. 首次显示时加载资源 + 绑定按钮
    abstract void OnLoadResouse(T self, string szWindowName, Dictionary<EUILayer, Transform> dicLayer);

    // 3. 窗口销毁时释放资源
    abstract void OnDestroyResouse(T self);

    // 4. 窗口每次显示时触发（刷新数据）
    abstract void OnShowWindow(T self, object showData);

    // 5. 窗口隐藏时触发（清理临时数据）
    abstract void OnHideWindow(T self);
}
```

**生命周期执行顺序**：

```
系统启动
  ↓ 自动扫描所有 [UIEvent] 类
OnCreateWindow → 在 UIComponent 上 AddComponent<T>()
  ↓ 调用 ShowWindow("XXX")
OnLoadResouse  → 加载 AssetBundle → Instantiate → 绑按钮
  ↓
OnShowWindow   → 传递数据、刷新显示
  ↓ 调用 HideWindow("XXX")
OnHideWindow   → 隐藏窗口、清理状态
  ↓ 调用 HideWindowDestroy("XXX")
OnDestroyResouse → Destroy 预制体
```

### 2.4 [UIEvent] 特性

每个窗口事件类通过 `[UIEventAttribute(nameof(XXX))]` 标记，系统启动时自动扫描并注册：

```csharp
[UIEventAttribute(nameof(DlgRoleUp))]  // nameof = "DlgRoleUp"
public class DlgRoleUpUIEvent : UIEventBase<DlgRoleUp>
```

注册流程（UIComponent.Awake）：
1. 扫描所有带 `[UIEvent]` 的类
2. 实例化并存入 `m_dicEvent["DlgRoleUp"]`
3. 调用 `CreateWindow()` → `AddComponent<DlgRoleUp>()`

## 三、窗口开发完整示例

以 `DlgRoleUp`（升级对话框）为例，展示完整的三层实现：

### 3.1 ModelView 层：DlgRoleUp.cs

```csharp
[ComponentOf(typeof(UIComponent))]
public class DlgRoleUp : Entity, IAwake, IDestroy
{
    public GameObject m_window = null;  // 预制体实例引用
}
```

职责仅：定义 Entity 类型，持有窗口 GameObject 引用。

### 3.2 HotfixView 层：DlgRoleUpEvent.cs

```csharp
[FriendClass(typeof(DlgRoleUp))]
[UIEventAttribute(nameof(DlgRoleUp))]
public class DlgRoleUpUIEvent : UIEventBase<DlgRoleUp>
{
    // 1. 注册窗口组件
    public override void OnCreateWindow(ref UIComponent pUIComponent)
        => pUIComponent.AddComponent<DlgRoleUp>();

    // 2. 加载预制体 + 绑定按钮
    public override void OnLoadResouse(DlgRoleUp self, string szWindowName, Dictionary<EUILayer, Transform> dicLayer)
    {
        // 加载 AssetBundle
        self.Domain.GetComponent<ResourcesLoaderComponent>().Load("UILogin.unity3d");
        // 从 Bundle 中获取预制体
        GameObject prefab = (GameObject)ResourcesComponent.Instance.GetAsset("UILogin.unity3d", "dlg_role_up");
        // 实例化到指定层级
        self.m_window = GameObject.Instantiate(prefab, dicLayer[EUILayer.high]);
        self.m_window.name = szWindowName;
        // 注册按钮事件
        UITools.FindChildComponent<Button>(self.m_window, "EBt_Back")
            ?.AddListener(self.OnClickClose);
        UITools.FindChildComponent<Button>(self.m_window, "EBt_LevelUp")
            ?.AddListenerAsync(self.OnClickLevelUp);
    }

    // 3. 销毁时释放
    public override void OnDestroyResouse(DlgRoleUp self) { ... }

    // 4. 显示时刷新
    public override void OnShowWindow(DlgRoleUp self, object showData)
        => self.OnRefresh();

    // 5. 隐藏时清理
    public override void OnHideWindow(DlgRoleUp self) { }
}
```

### 3.3 HotfixView 层：DlgRoleUpSystem.cs

```csharp
[FriendClassAttribute(typeof(DlgRoleUp))]
[FriendClassAttribute(typeof(PlayerRoleUpComponent))]
public static class DlgRoleUpSystem

    // 升级按钮 - 异步RPC
    public static async ETTask OnClickLevelUp(this DlgRoleUp self) { ... }
}
```

## 四、资源加载机制

### 4.1 AssetBundle 加载方式

```
LoadBundle("uilogin.unity3d")
  → 加载 .unity3d 文件
  → LoadAllAssets() 解析所有资源
  → 缓存到 resourceCache 字典
  ↓
GetAsset("uilogin.unity3d", "dlg_role_up")
  → 从缓存中取出预制体
```

### 4.2 ResourcesLoaderComponent（引用计数）

```csharp
// 加载（增加引用计数）
self.Domain.GetComponent<ResourcesLoaderComponent>().Load("UILogin.unity3d");

// 内部实现：如果已加载则跳过，否则加载并记录
public static void Load(this ResourcesLoaderComponent self, string ab)
{
    if (self.LoadedResource.Contains(ab)) return;
    self.LoadedResource.Add(ab);
    ResourcesComponent.Instance.LoadBundle(ab);
}
```

### 4.3 预制体资源路径规范

```
Assets/Bundles/
├── UI/                    ← 主界面预制体（ui*.unity3d）
│   └── UILobby.prefab     → uilobby.unity3d
├── dialog/                ← 弹窗预制体（共用 uilogin.unity3d）
│   ├── dlg_common/        ← 通用控件
│   ├── dlg_login/         ← 登录界面
│   ├── dlg_package/       ← 背包界面
│   ├── dlg_gm/            ← GM工具
│   ├── dlg_role/          ← 角色升级
│   │   └── dlg/dlg_role_up.prefab → uilogin.unity3d
│   ├── dlg_updata/        ← 更新界面
│   └── dlg_reconnection/  ← 重连界面
├── Config/                ← 配置资源
├── Code/                  ← 热更代码资源
├── sound/                 ← 音效
└── resources/
```

**关键规则**：所有 `dialog/` 目录下的预制体打包到同一个 `uilogin.unity3d`，主界面各自独立打包（如 `uilobby.unity3d`）。

## 五、UI 辅助工具

### 5.1 UITools

| 方法 | 说明 |
|---|---|
| `UITools.Tips(string)` | 弹出 Toast 提示（打开 UITips 窗口） |
| `FindChildComponent<T>(parent, name, searchType)` | 按名称查找子节点组件（深度/广度搜索） |
| `FindChild(parent, name, searchType)` | 按名称查找子节点 |
| `DestroyChildren(trans)` | 销毁所有子对象 |

### 5.2 UIHelper（EUIHelper）

| 方法 | 说明 |
|---|---|
| `button.AddListener(action)` | 同步按钮事件 |
| `button.AddListenerAsync(asyncAction)` | 异步按钮事件（带 UI 锁） |
| `button.AddListenerWithId(action, id)` | 按钮事件带 int 参数 |
| `button.AddListenerWithParam(action, param)` | 按钮事件带泛型参数 |
| `text.SetText(content)` | 安全设置文本 |
| `image.SetSprite(path)` | 设置图片 |

### 5.3 UI 锁机制

异步按钮事件自动加锁，防止连续点击：

```csharp
async ETTask clickActionAsync()
{
    UIComponent.Instance?.AddLock();   // 锁定 UI
    await action();
    UIComponent.Instance?.UnLock();     // 解锁
}
```

## 七、列表 Item 缓存机制

列表界面通过 `ItemHold/ItemGet` 实现 Item 复用，避免重复实例化：

```csharp
// Event 层：加载时缓存 Item 模板
GameObject pItem = UITools.FindChild(pE_SL_MaterialList, "Item_dlgpackagemain_MaterialList");
UIComponent.Instance.ItemHold(pItem);  // 缓存 Item

// System 层：从缓存获取并实例化填充
GameObject objItem = UIComponent.Instance.ItemGet("Item_dlgpackagemain_MaterialList");
GameObject pItem = GameObject.Instantiate(objItem, pContent.transform);
UITools.FindChildComponent<Text>(pItem, "ELb_MaterialName")?.SetText(pMaterialConfig.Name);
```

## 八、编写新窗口的步骤

```
1. 创建预制体
   Assets/Bundles/dialog/xxx/ 下放置 .prefab
   设置 AssetBundle 名 → 通常用 uilogin.unity3d

2. 创建 ModelView 文件
   Codes/ModelView/Client/UIXXX/DlgXXX.cs
   → 继承 Entity, 声明 GameObject m_window

3. 创建 Event 文件
   Codes/HotfixView/Client/UIXXX/DlgXXXEvent.cs
   → 继承 UIEventBase<DlgXXX>
   → 实现 5 个生命周期方法

4. 创建 System 文件
   Codes/HotfixView/Client/UIXXX/DlgXXXSystem.cs
   → 按钮回调（异步/同步）
   → 刷新逻辑

5. 打开窗口
   UIComponent.Instance.ShowWindow("DlgXXX");
```
